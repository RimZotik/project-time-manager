# Project Time Manager

[README in English](README.en.md)

**Project Time Manager** — настольный трекер времени для Windows. Он считает не просто «время за компьютером», а **реально затраченное время на конкретный проект**: какие приложения, вкладки браузера и сайты были активны, по каким этапам шла работа — чтобы потом смотреть красивую аналитику и отчёты прямо в приложении.

> Приложение работает на Windows (реальное отслеживание активного окна — через WinAPI). На macOS/Linux открывается интерфейс для разработки, но трекинг активных окон недоступен.

## Возможности

- **Проекты** с категориями (например, Монтаж, Программирование) — цвет, иконка, фильтр по категориям.
- **Запись времени**: старт / пауза / стоп. Индикатор записи всегда виден в правом верхнем углу.
- **Отслеживание активности** (Windows): активное окно, приложения с реальными иконками, браузеры с раскрытием до сайтов и списком посещённых ссылок.
- **Включение / исключение** приложений, сайтов и ссылок из итога — данные не удаляются, просто не учитываются.
- **Этапы проекта**: выбираются перед стартом сессии, время распределяется по этапам.
- **Отчёты с таймлайном** — как в монтажных программах: дорожки по этапам и по сессиям, зум колесом, прокрутка, перетаскивание.
- **Аналитика** по всем проектам: время по проектам, по категориям (donut), активность по дням, тепловая карта «часы × дни недели», топ приложений.
- **Дашборд**: живой статус записи с таймером, KPI, топ-проекты, активность за неделю, быстрый старт.
- **Мониторинг**: активное окно сейчас, таймер сессии, живой список приложений.
- **Автокатегоризация**: правила «приложение → категория» и кнопка «Определить категорию по приложениям» на проекте.
- **Автобэкап** базы данных при запуске (хранятся последние 10 копий).
- **Настройки**: язык (русский / English), автозапуск с Windows, папка данных.

## Страницы

| Раздел | Что показывает |
|---|---|
| **Дашборд** | Обзор по всем проектам, статус записи, быстрый старт |
| **Проекты** | Список с категориями + детали проекта во вкладках: Обзор · Приложения и вкладки · Этапы · Настройки проекта · Отчёты (таймлайн) |
| **Аналитика** | Графики и сравнение по всем проектам и категориям |
| **Мониторинг** | Живая активность: активное окно, таймер, приложения |
| **Настройки** | Язык, автозапуск, правила автокатегоризации, папка данных |

## Данные

Данные хранятся в SQLite-базе рядом с исполняемым файлом:

```text
data/
  ptm.db            # основная база (проекты, сессии, приложения, вкладки, ссылки, этапы, категории)
  backups/          # автоматические резервные копии (последние 10)
```

При первом запуске, если рядом лежат старые JSON-данные (`data/workspace.json` и `data/Проекты/<Имя>/project.json` из ранних версий), они **автоматически мигрируют** в новую базу.

## Установка (Windows)

1. Скачайте установщик `project-time-manager.exe` из последнего [релиза](../../releases/latest).
2. Запустите — приложение установится для текущего пользователя.
3. При переходе со старой версии положите папку `data` со старыми проектами рядом с приложением до первого запуска — данные мигрируют сами.

## Стек

- [Tauri 2](https://tauri.app/) + Rust (бэкенд)
- React 19 + TypeScript + Vite + Tailwind CSS 4 (фронтенд)
- SQLite через `rusqlite` (хранилище)
- WinAPI через crate `windows` (отслеживание активного окна)
- `framer-motion` (анимации)

## Разработка

```bash
npm install
npm run tauri:dev            # запуск в режиме разработки
```

```bash
npm run build                # проверка типов + сборка фронтенда
cd src-tauri && cargo test   # тесты бэкенда (миграция, хранилище, категории)
```

На macOS/Linux интерфейс откроется, но отслеживание активных окон рассчитано только на Windows.

## Сборка и релиз

Сборка Windows-установщика и публикация релиза выполняются автоматически через GitHub Actions.

1. Обновите версию в `package.json`, `src-tauri/Cargo.toml` и `src-tauri/tauri.conf.json`.
2. Создайте и запушьте тег `vX.Y.Z`:

   ```bash
   git tag v0.2.0
   git push origin main --tags
   ```

3. Workflow [`.github/workflows/windows-build.yml`](.github/workflows/windows-build.yml) соберёт NSIS-установщик и создаст GitHub Release с файлом `project-time-manager.exe`.

Ручной запуск сборки — через вкладку **Actions → Windows Build → Run workflow**.

## Структура кода

```text
src/
  App.tsx                     # роутинг + оболочка
  components/layout/          # боковое меню, топбар, фон с волнами

  lib/                        # типы, api, i18n

  main.rs        # Tauri-команды, трекинг, запуск
  db.rs          # SQLite: схема, миграция, запросы, аналитика
  storage.rs     # слой хранения поверх SQLite, бэкап, категории, правила
  windows.rs     # WinAPI: активное окно, процесс, URL/иконки
  models.rs      # модели данных
```
